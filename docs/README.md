## Documentation Standards

### What Goes Here (`docs/`)
- **Active setup guides** that users/developers follow
- **Reference documentation** that is actively maintained and consulted
- **Operational guides** with pending/ongoing tasks
- **Technical specifications** that evolve with the codebase

### What Goes in `docs-archive/`
- Completed feature implementations
- Historical migration documentation
- Point-in-time audits and reviews
- Superseded proposals and plans

### What Stays at Root Level
- `README.md` - Project overview
- `CLAUDE.md` - AI assistant guidance
- `FEATURES.md` - Feature catalog
- `TODO.md` - Task tracking
- `STRATEGIC_ENHANCEMENTS.md` - Future roadmap
